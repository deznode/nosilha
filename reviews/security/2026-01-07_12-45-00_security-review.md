### File Upload Security (Gallery Module)

**Files Reviewed:**
- `apps/api/src/main/kotlin/com/nosilha/core/gallery/api/GalleryController.kt`
- `apps/api/src/main/kotlin/com/nosilha/core/gallery/domain/R2StorageService.kt`
- `apps/api/src/main/kotlin/com/nosilha/core/gallery/domain/GalleryService.kt`

**Assessment:** SECURE
- Content-Type validation with strict whitelist: `^(image/(jpeg|png|webp|gif)|video/mp4)$`
- File size limit enforced (50MB max via `@Max` annotation)
- Filename sanitization removes dangerous characters
- Storage keys use UUID prefix preventing predictable paths
- Bucket4j rate limiting (20/hour, 100/day per user)

### Path Traversal Protection (Stories Module)

**Files Reviewed:**
- `apps/api/src/main/kotlin/com/nosilha/core/stories/services/MdxFileWriter.kt`
- `apps/api/src/main/kotlin/com/nosilha/core/stories/api/AdminMdxController.kt`

**Assessment:** SECURE
- Multi-layered protection against path traversal:
  - Slug whitelist validation: `^[a-z0-9]+(?:-[a-z0-9]+)*$`
  - Path normalization via `toAbsolutePath().normalize()`
  - Containment check: `targetPath.startsWith(basePath)`
- YAML deserialization uses `SafeConstructor` (prevents CVE-2022-1471)
- Alias limit set to prevent billion laughs attacks

**Location:** `apps/web/src/app/actions/archive-story.ts:83`

**Initial Concern:** The `slug` parameter is used in GitHub file path construction without frontend validation.

**Determination:** FALSE POSITIVE

**Reasons for Exclusion:**
1. The slug originates from the backend database, not direct user input
2. Backend validates slugs with strict whitelist regex `^[a-z0-9]+(?:-[a-z0-9]+)*$`
3. Admin authentication required to access this server action
4. GitHub API provides additional path validation/normalization
5. This is a defense-in-depth recommendation, not an exploitable vulnerability
