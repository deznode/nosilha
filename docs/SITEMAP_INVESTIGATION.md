# Sitemap Investigation - Production 500 Error

**Date:** 2025-11-18
**Status:** 🔴 Failing in Production (500 Error)
**Priority:** Medium - Blocking Google Search Console submission

## Summary

The Nos Ilha platform has a **comprehensive dynamic sitemap** implemented at `frontend/src/app/sitemap.ts`, but it's currently returning a **500 Internal Server Error** in production at `https://nosilha.com/sitemap.xml`.

## Current Implementation

### Location
- **File:** `frontend/src/app/sitemap.ts`
- **Route:** `/sitemap.xml` (auto-generated by Next.js)
- **Build Status:** ✅ Builds successfully locally
- **Production Status:** ❌ Returns 500 error

## Problem Analysis

### Root Cause (Suspected)

The sitemap generation calls the backend API during build/runtime:

```typescript
// Line 125 in sitemap.ts
const { items: allEntries } = await getEntriesByCategory("all");
```

**Issue:** This API call is failing in production, causing a 500 error.

### Build Evidence

Local build output shows successful generation:
```bash
├ ○ /sitemap.xml    147 B    103 kB    1h    1y
```

However, API connection errors appear during build:
```
[TypeError: fetch failed] {
  [cause]: [AggregateError: ] { code: 'ECONNREFUSED' }
}
```

### Fallback Mechanism

The code **does have** a try/catch fallback:

```typescript
try {
  // Fetch all directory entries for dynamic pages
  const { items: allEntries } = await getEntriesByCategory("all");
  // ... generate dynamic pages
  return [...staticPages, ...dynamicPages, ...townPages];
} catch (error) {
  console.error("Error generating sitemap:", error);
  // Return static pages if dynamic content fails
  return staticPages;
}
```

**Expected Behavior:** Should return static pages (19 routes) if API fails
**Actual Behavior:** Returns 500 error in production

## Possible Causes

### 1. Build-Time vs Runtime API Call

**Hypothesis:** Next.js may be trying to fetch data at **build time** when the backend API is unreachable.

**Evidence:**
- Build logs show `ECONNREFUSED` errors
- Sitemap generation happens during static generation phase
- Backend API may not be accessible from build environment

### 2. Production Environment Variables

**Hypothesis:** `NEXT_PUBLIC_API_URL` may be misconfigured for sitemap generation.

**Check Required:**
- Verify `siteConfig.url` resolves to correct production URL
- Verify backend API is accessible from Cloud Run frontend container
- Check if API calls work in other server-side routes

### 3. CORS or Network Issues

**Hypothesis:** Network/CORS blocking API calls during sitemap generation.

**Check Required:**
- Verify backend CORS allows frontend domain
- Check if backend is accessible from frontend Cloud Run service
- Review Cloud Run service-to-service communication

### 4. ISR/Dynamic Route Configuration

**Hypothesis:** Next.js ISR configuration may be interfering with sitemap generation.

**Check Required:**
- Review `next.config.ts` for sitemap-related configuration
- Check if route segment config affects sitemap

### Phase 3: Solutions (Based on Findings)

#### Solution A: Static-Only Sitemap (Quick Fix)

**If:** API will always be unreachable during build

**Then:** Modify `sitemap.ts` to return only static pages:
```typescript
export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  const baseUrl = siteConfig.url;
  const currentDate = new Date().toISOString();

  // Return static pages only (remove API call)
  return staticPages;
}
```

**Pros:**
- ✅ Immediate fix for Google Search Console
- ✅ No API dependency
- ✅ Reliable and predictable

**Cons:**
- ❌ Missing dynamic entries (directory items, towns)
- ❌ Less comprehensive SEO coverage

#### Solution B: Enhanced Error Handling

**If:** API should be reachable but isn't due to configuration

**Then:** Add better error handling and logging:
```typescript
try {
  console.log('[Sitemap] Attempting to fetch entries from:', process.env.NEXT_PUBLIC_API_URL);
  const { items: allEntries } = await getEntriesByCategory("all");
  console.log('[Sitemap] Successfully fetched', allEntries.length, 'entries');
  // ... generate dynamic pages
} catch (error) {
  console.error('[Sitemap] Failed to fetch entries:', error);
  console.log('[Sitemap] Falling back to static pages only');
  return staticPages;
}
```

#### Solution C: Separate Static Build Step

**If:** We want dynamic content but build-time fetch is unreliable

**Then:** Use Next.js ISR to regenerate sitemap periodically:
```typescript
export const revalidate = 3600; // Regenerate every hour

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  // This will regenerate on-demand in production
  // ...
}
```

#### Solution D: Pre-Build Sitemap Generation

**If:** We want comprehensive sitemap with dynamic content

**Then:** Generate sitemap as a separate CI/CD step:
1. Add script to fetch data and generate sitemap.xml
2. Run script during GitHub Actions build
3. Include static sitemap.xml in build output
